# File Map

A concise purpose statement for every file in the repository, plus notable exports and relationships. Paths are clickable.

## Root
- App.tsx: application entry and simulation loop; orchestrates physics ticks, predictions, camera, UI panels, and the Flight Computer pipeline.
- index.tsx: bootstraps React and mounts <App />.
- index.html: Vite HTML template, import map for CDN dependencies; Tailwind via CDN for styling.
- vite.config.ts: Vite config; defines `@/` alias and injects `process.env.GEMINI_API_KEY`.
- tsconfig.json: TypeScript config (ES2022, react-jsx, bundler resolution, paths alias).
- package.json: scripts (dev/build/preview) and dependencies (react, drei/fiber, three, @google/genai, recharts, dnd-kit).
- metadata.json: app metadata used by AI Studio.
- README.md: run instructions (local dev) generated by AI Studio.
- AGENTS.md: conventions and architecture guide for this repo.
- constants.ts: default configs, helper `createBody`, and many presets (Solar, Binary, Figure-8, hierarchical triples, etc.).
- types.ts: canonical TypeScript interfaces (Body, Maneuver, VisualConfig, PhysicsConfig, FlightComputer* types, etc.).

## Components (`components/`)
- Assistant.tsx: ŌĆ£CosmosŌĆØ AI chat UI. Sends text and optional voice input to Gemini; handles function-calling tools through `services/geminiService.ts`; optional TTS playback/visualizer.
- BuilderPanel.tsx: panel to create a new body (name, mass, distance, velocity, color). Emits `onAddBody` with normalized radius.
- Canvas.tsx: 2D renderer (HTML5 Canvas). Draws background (stars/nebula), gravity grid, trails, bodies, rockets, CoM, particles, flight-computer overlays, rendezvous markers; handles pan/zoom, selection, touch gestures.
- Canvas3D.tsx: 3D renderer (Three.js via @react-three/fiber). Bodies as meshes with glow/atmosphere; optional trails/overlays; OrbitControls; gravity grid in 3D; rendezvous and module visualizations.
- CoMInfoPanel.tsx: small panel showing real/refined center-of-mass values, threshold and included/excluded bodies.
- Controls.tsx: bottom control bar (play/pause, reset, reverse time, speed, zoom, toggles for grid/observer/assistant/CoM/rocket, preset chooser). Mobile/desktop layouts.
- FlightComputerPanel.tsx: UI editor/executor for modular ŌĆ£Flight ComputerŌĆØ system. Adds/removes/toggles modules, groups, inputs/outputs wiring, sound/notify/logic, thrust bursts, maneuver executor, marker, rendezvous tools.
- GravityObserverPanel.tsx: analysis overlay for pairwise gravity between two bodies; net forces, velocities, and sparkline history (Recharts).
- InfoPanel.tsx: object inspector for selected body: description, mass/diameter/period/current velocity and comparison chart.
- ManualCreationPanel.tsx: minimal overlay for manual body placement, with live prediction steps and basic body properties editing.
- MusicPanel.tsx: experimental AI music generator (Google GenAI live music). Manages weighted text prompts, streaming audio, and an effects chain (reverb/lowpass) in Web Audio.
- PredictionPanel.tsx: top bar to enable/disable trajectory prediction, tune steps, select bodies to trace, and follow a body.
- RocketDataPanel.tsx: compact rocket HUD (speed, heading, fuel, orbital parameters, closest approach; optional phase-angle tracker) with prediction-derived timing.
- RocketPanel.tsx: full rocket control and mission planner. Manual thrust/rotate, SAS, auto maneuvers (circularize, land, transfer, intercept), flight recorder, manual nodes, queue execution, spawn UI.
- SettingsPanel.tsx: consolidated settings for visuals, physics, save/load, 3D switch, AI API key, audio status, and music panel toggle.

## Hooks (`hooks/`)
- useIsMobile.ts: viewport width detector for conditional UIs.
- useRocketSound.ts: Web Audio engine for engine noise + event beeps (maneuver starts, SAS changes, low-altitude warnings, land/crash SFX). Lazy initialization and iOS silent-switch bypass.

## Services (`services/`)
- physicsEngine.ts: core deterministic physics and mission mechanics.
  - Forces, sub-stepping integrator, collisions/merging/landing, particle effects, time-reverse.
  - Maneuver execution (burns, wait, closed-loop circularize, wait-for-altitude/transfer, burn-until-altitude, auto-transfer/intercept via Lambert solver).
  - Helpers: `calculateForces`, `updatePhysics`, `calculateOrbitalManeuver`, `predictSystemTrajectories`, `calculateOrbitalPoints`, `calculateEllipsePoints`, `solveLambert`, `reverseTime`.
- orbitalMath.ts: orbital analytics + typed input resolution for modules.
  - `calculateOrbitInfo` (apoapsis/periapsis/period/altitude, Pe/Ap points), `calculateTransferInfo` (Hohmann phases, wait/transfer/arrival times and intercept points).
  - Flight Computer input resolvers: `resolveInput`, `resolveScalarInput`, `resolveBooleanInput`, `resolveStringInput`.
- geminiService.ts: Gemini client and function-call tools definitions used by Assistant.
  - Tools: spawn/delete/make_star, control simulation/camera/visuals/physics, rocket control, program/execute plans, flight computer add/toggle/remove/get, telemetry, manual node.
  - `createChatSession` wires tools + system instructions; `generateSpeech` TTS.
- predictionWorker.ts: Web Worker to compute multi-step predictions off the main thread using `calculateForces`.

## Utils (`utils/`)
- audioUtils.ts: base64 encode/decode, PCM conversions, and AudioBuffer creation helpers for streaming audio from AI.
